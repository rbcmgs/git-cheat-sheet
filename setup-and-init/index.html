    Setting up Git and Initializing RepositoriesÂ¶

    A guide to getting started with Git, from installation to creating your first repository.

    Installing GitÂ¶

    WindowsÂ¶

    Official InstallerÂ¶

    Download from https://git-scm.com/download/win and run the installer with default options.

    Using WingetÂ¶

    winget install --id Git.Git

    Using ChocolateyÂ¶

    choco install git

    macOSÂ¶

    Using HomebrewÂ¶

    brew install git

    Using the official installerÂ¶

    Download from https://git-scm.com/download/mac and follow installation instructions.

    LinuxÂ¶

    Debian/UbuntuÂ¶

    sudo apt-get update

    sudo apt-get install git

    FedoraÂ¶

    sudo dnf install git

    RHEL/CentOSÂ¶

    sudo yum install git

    Arch LinuxÂ¶

    sudo pacman -S git

    Verify installationÂ¶

    git --version

    Initial ConfigurationÂ¶

    Setting Your IdentityÂ¶

    Git requires knowing who you are for commit attribution:

    Set your nameÂ¶

    git config --global user.name "Your Name"

    Set your emailÂ¶

    git config --global user.email "your.email@example.com"

    ðŸ’¡ Tip: Use the same email address you use for your remote Git hosting service (GitHub, GitLab, etc.).

    Configuring Your EditorÂ¶

    Set your preferred text editor for commit messages:

    For VS CodeÂ¶

    git config --global core.editor "code --wait"

    For VimÂ¶

    git config --global core.editor "vim"

    For Notepad++ (Windows)Â¶

    git config --global core.editor "'C:/Program Files/Notepad++/notepad++.exe' -multiInst -notabbar -nosession -noPlugin"

    For Sublime TextÂ¶

    git config --global core.editor "'subl' -w"

    Checking Your SettingsÂ¶

    Review all your settings:

    List all settingsÂ¶

    git config --list

    Check a specific settingÂ¶

    git config user.name

    git config user.email

    Configuration LevelsÂ¶

    Git configurations can be applied at three levels:

    System levelÂ¶

    Applies to all users:

    git config --system ...

    Global levelÂ¶

    Applies to all repositories of current user:

    git config --global ...

    Local levelÂ¶

    Applies only to current repository:

    git config --local ...

    ðŸ“Œ Note: Local settings override global settings, which override system settings.

    Creating RepositoriesÂ¶

    Initializing a New RepositoryÂ¶

    Turn an existing directory into a Git repository:

    Navigate to your project directoryÂ¶

    cd path/to/your/project

    Initialize the repositoryÂ¶

    git init

    After initializing, you'll see a .git directory which contains all the Git metadata.

    Cloning an Existing RepositoryÂ¶

    Copy a repository from a remote server:

    Clone via HTTPSÂ¶

    git clone https://github.com/username/repository.git

    Clone via SSHÂ¶

    Requires setup of SSH keys:

    git clone git@github.com:username/repository.git

    Clone to a specific directoryÂ¶

    git clone https://github.com/username/repository.git my-project

    Clone a specific branchÂ¶

    git clone -b branch-name https://github.com/username/repository.git

    Clone with limited historyÂ¶

    Shallow clone:

    git clone --depth=1 https://github.com/username/repository.git

    First-time Repository SetupÂ¶

    Standard Project FilesÂ¶

    When creating a new repository, consider adding these standard files:

    Create a README fileÂ¶

    echo "# Project Name" > README.md

    Create a license fileÂ¶

    Example: MIT

    curl -o LICENSE https://opensource.org/licenses/MIT

    Create a .gitignore fileÂ¶

    Discussed below:

    touch .gitignore

    Creating a .gitignore FileÂ¶

    The .gitignore file specifies intentionally untracked files that Git should ignore:

    touch .gitignore

    Sample .gitignore content for various project types:

    Node.jsÂ¶

    node_modules/
npm-debug.log
.env

    PythonÂ¶

    __pycache__/
*.py[cod]
*$py.class
venv/
.env

    JavaÂ¶

    *.class
*.jar
target/
.idea/

    ðŸ’¡ Tip: Use gitignore.io to generate .gitignore files for your specific tech stack.
